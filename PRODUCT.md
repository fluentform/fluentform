# Fluent Forms Product Context

## Product

Fluent Forms is a WordPress form builder and form operations plugin. Its admin UI lets site owners and operators build forms, manage submissions, configure notifications and integrations, review reports, and handle payments.

## Register

product

## Primary Users

- WordPress site owners who need to create and maintain forms without engineering help.
- Agency builders who manage many forms across client sites and need predictable admin workflows.
- Support and operations users who review entries, triage submissions, export data, and inspect integration activity.
- Store, event, education, and lead-generation teams that rely on forms as an operational inbox.

## Product Purpose

Help users create forms quickly, collect submissions reliably, and manage follow-up workflows inside WordPress admin. The UI should make repeated operational tasks faster: scanning entries, filtering lists, opening details, exporting data, configuring integrations, and confirming settings.

## Design Principles

- Task-first product UI. Entries, settings, and integrations are working surfaces, not marketing surfaces.
- Keep important state visible: active filters, selected form, current status, loading, empty, and permission states.
- Prefer familiar WordPress/admin conventions over novel visual patterns.
- Make tables dense but readable. Avoid adding duplicate count labels when pagination already owns totals.
- Keep destructive actions explicit and recoverable.
- Use concise copy. Avoid paragraphs in dense admin tables unless the user is blocked.
- Optimize for repeat use on laptops and narrow WordPress admin widths.

## Tone

Clear, calm, operational, and confident. Use plain labels and action-oriented empty states. Avoid hype, jokes, decorative language, and marketing-style value props in admin screens.

## Anti-References

- Marketing landing-page composition inside admin pages.
- Oversized hero cards or ornamental empty states for routine tools.
- Redundant helper text that repeats visible controls.
- Icon-only actions without accessible names or tooltips.
- Hidden filtered states that make counts look wrong.
- Table layouts that force users to scroll horizontally before seeing core entry information.

## Key Screens

- Form builder editor.
- Form-specific entries.
- Global entries.
- Entry detail.
- Form settings and integrations.
- Global settings and integration modules.
- Reports and payments.
